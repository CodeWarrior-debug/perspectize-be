       ```markdown
       ## Responsive Breakpoints

       This project uses Tailwind's mobile-first breakpoint system:

       | Breakpoint | Min Width | Target Devices | Layout Behavior |
       |------------|-----------|----------------|-----------------|
       | (base) | 0px | iPhone SE, small phones | Single column, px-4 padding |
       | sm: | 640px | Large phones, small tablets | Minor adjustments only |
       | md: | 768px | Tablets, small laptops | px-6 padding, 2-column options |
       | lg: | 1024px | Desktops, large tablets | px-8 padding, full features |
       | xl: | 1280px | Large desktops | max-w-screen-xl centered |
       | 2xl: | 1536px | Extra large screens | Same as xl (no further scaling) |

       ### Key Principles

       1. **Mobile-first**: Base styles (no prefix) target mobile. Add prefixes to enhance for larger screens.
       2. **iPhone SE minimum**: 375px is our smallest target. Test at this width.
       3. **Max width constraint**: Content maxes out at 1280px (max-w-screen-xl) to maintain readability.
       4. **Consistent padding**: px-4 (mobile) -> px-6 (tablet) -> px-8 (desktop)

       ### Testing Checklist

       When creating responsive components, test at these widths:
       - 375px (iPhone SE)
       - 768px (iPad portrait)
       - 1024px (iPad landscape / small desktop)
       - 1280px+ (desktop)
       ```

    2. Update test page to show current viewport width for testing:
       Add a small debug display (can be removed later):
       ```svelte

         import { browser } from '$app/environment';

           if (browser) {

       </script>
       ```
